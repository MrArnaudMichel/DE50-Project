Cahier des charges - Plugin Rhapsody SVN

1. Introduction et contexte

Ce document décrit les spécifications d'un plugin pour IBM Rhapsody permettant de modéliser des diagrammes de type Stakeholder Value Network (SVN).

Le graphique SVN est issu des travaux de Cameron (2007) et a été démontrée dans le cadre SysML par Sease et al. (INCOSE 2018) :

  • Setting priorities: demonstrating Stakeholder Value Networks in SysML – INCOSE (https://incose.org/resource/setting-priorities-demonstrating-stakeholder-value-networks-in-sysml)

  • Publication ResearchGate (https://researchgate.net/publication/327071403)

2. À qui s'adresse le plugin

2.1 Profil type

Le plugin s'adresse à toute personne amenée à concevoir ou analyser un système complexe dans un contexte professionnel ou académique. Il cible en priorité les praticiens de l'ingénierie systèmes qui utilisent déjà IBM Rhapsody comme outil de modélisation au quotidien et qui souhaitent enrichir leur démarche avec une analyse structurée de la valeur perçue par les différents acteurs du système.

Informations

L'utilisateur N'a pas besoin d'être expert des graphiques SVN pour utiliser le plugin. Une connaissance de base de SysML et de Rhapsody est suffisante pour démarrer.

2.2 Contexte d'utilisation

Le plugin est conçu pour être utilisé en phase amont d'un projet, lors de la définition de l'architecture système. C'est à ce stade que les choix structurants sont faits et que disposer d'une vision claire et quantifiée des priorités du système apporte le plus de valeur.

Il s'intègre naturellement dans une démarche MBSE (Model-Based Systems Engineering), où toutes les décisions d'architecture sont portées par le modèle.

2.3 Objectifs poursuivis par l'utilisateur

  • Avoir une vision globale et structurée du système qu'il conçoit ou analyse

  • Prioriser les orientations de conception de manière objective et justifiable

  • Détecter en amont les conflits potentiels entre les différents objectifs du système

  • Gagner du temps sur les calculs d'analyse en les automatisant

  • Disposer de résultats directement intégrés dans le modèle Rhapsody, sans ressaisie

2.4 Attentes vis-à-vis du plugin

  • Prise en main rapide, sans formation approfondie sur les SVN

  • Interface et ergonomie cohérentes avec Rhapsody

  • Calculs automatiques et résultats lisibles directement dans le modèle

  • Visualisation intuitive avec mise en évidence des éléments prioritaires

  • Traçabilité des résultats vers les autres éléments du modèle (exigences, fonctions)

3. Objectifs du diagramme SVN

3.1 Modéliser la structure de valeur d'un système

Le diagramme permet de représenter graphiquement un système et l'ensemble des acteurs qui gravitent autour de lui. Chaque relation entre un acteur et le système, ou entre deux acteurs, est modélisée comme un flux de valeur orienté et pondéré. L'ensemble de ces flux forme un réseau qui reflète la réalité des échanges et des dépendances du système.

3.2 Quantifier l'influence des acteurs

Sur la base du réseau modélisé, le diagramme permet de calculer automatiquement l'importance relative de chaque acteur. Ce calcul repose sur les fomrules relatives aux SVN (Cameron, 2007) et les équations formalisées dans la recherche INCOSE 2018. Il produit un classement objectif qui pourra servir de support dans tout type de démarche de développement.

3.3 Informer les choix d'architecture

Les résultats issus du diagramme SVN ont vocation à se propager dans le reste du modèle système. Ils permettent de hiérarchiser les exigences, de pondérer les objectifs système et d'identifier les conflits potentiels entre fonctions, en donnant à l'architecte une base quantitative pour arbitrer ses trade-offs.

Résumé

Le diagramme SVN poursuit trois objectifs complémentaires :

  • Représenter graphiquement les flux de valeur entre les acteurs d'un système

  • Calculer automatiquement l'importance relative de chaque acteur

  • Propager ces résultats dans le modèle système pour informer les choix d'architecture et arbitrer les trade-offs.

Ces trois axes sont les points centraux ajoutés par ce plugin.

4. Fonctions du plugin

4.1 Gestion des éléments du graphe

4.1.1 Liste des éléments disponibles et dessinables

  • Stakeholder

  • Système

  • Arcs

4.1.2 Fonctions possibles sur ces éléments

  • Ajouter et supprimer des éléments sur le graphe

  • Modifier les propriétés d'un élément existant :

    • nom

    • description

    • pondération (arcs)

    • couleur (en fonction de l'élément)

  • Changer d'échelle dans l'hypergraphe

Attention

Un SVN est un hypergraphe. Il est donc possible de redéfinir à l'intérieur d'un stakeholder un SVN qui sera considéré comme une seule entité dans le graphe parent.

4.2 Calcul et classement des acteurs

  • Calculer automatiquement les scores de Value Path et Value Loop à partir des pondérations saisies

  • Classer les acteurs par ordre d'importance selon l'équation de priorisation (Cameron, Equation 2)

  • Mettre à jour le classement dynamiquement à chaque modification du graphe

  • Mettre à jour dynamiquement le style des éléments graphiques (couleur, graisse de la police, etc.)

  • Filtrer l'affichage pour ne montrer qu'un sous-ensemble d'acteurs ou de relations

  • Afficher ou masquer les pondérations des arcs selon le niveau de détail souhaité

Information

L'un des apports majeurs du plugin est d'automatiser les calculs définis par les SVN, qui sont fastidieux à réaliser manuellement. L'utilisateur saisit les données, le plugin produit ensuite le classement sans intervention supplémentaire. Ce classement pourra être affiché directement depuis l'interface utilisateur

4.3 Visualisation et code couleur

  • Mettre en évidence les Value Loops lors de la sélection d'un acteur

  • Afficher/cacher des éléments pour permettre à l'utilisateur de se concentrer sur le reste du graphe

6. Spécifications techniques

6.1 Approche technique générale

Le plugin est développé comme une extension des profils BDD (Block Definition Diagram) et IBD (Internal Block Diagram) déjà présents dans Rhapsody. Il ne crée pas un nouveau type de diagramme from scratch, mais enrichit les diagrammes existants avec les concepts propres aux graphique SVN.

Cette approche garantit :

  • La compatibilité avec les modèles Rhapsody existants

  • La cohérence visuelle et comportementale avec l'environnement natif

  • Une courbe d'apprentissage réduite pour l'utilisateur

6.2 Langage et API

Le plugin est développé en Java via l'API officielle d'IBM Rhapsody.

6.3 Extension des profils BDD et IBD

BDD (Block Definition Diagram)

Le BDD est étendu pour supporter la modélisation du graphe SVN :

  • Ajout du stéréotype «stakeholder» applicable aux blocs SysML

  • Ajout du stéréotype «svnSystem» pour identifier le système central de l'analyse

  • Ajout du stéréotype «valueArc» applicable aux associations, portant les tag values :

    • benefitRanking: MIGHT_BE/SHOULD_BE/MUST_BE

    • supplyImportance: LOW/MEDIUM/HIGH

IBD (Internal Block Diagram)

L'IBD est étendu pour représenter les flux de valeur entre instances :

  • Visualisation des flux orientés entre ports des blocs stakeholders

  • Affichage des pondérations sur les connecteurs

  • Mise en évidence des Value Loops sur sélection d'un élément
